定义类型与精简函数

定义新的数据类型

大多数语言都能定义新的数据类型,Haskell 也不例外.

data BookInfo = Book Int String [String] deriving (Show)

data是定义数据类型的关键字,BookInfo是类型构造器,即定义的类型是BookInfo.Book是值构造器,用于创建类型的实例.

Book之后有Int,String,[String]是类型的组成部分(域).BookInfo数据类型由ID(Int),书名(String),作者([String])组成.

注意:haskell要求类型构造子和值构造子必须首字母必须大写

类型的区分是通过类型构造器的,因此即使结构相同的类型,因为构造器的不同,它们也被视为不同的类型.

可以通过type查询值构造器的类型签名(类型构造器不行)

ghci>:type Book
Book :: Int -> String -> [String] -> BookInfo

类型构造器和值构造器

在Haskell中,类构造器和值构造器的名字是相互独立的,类型构造器只能出现在类型的定义中,或者类型签名中.而值构造器只能出现在实际的代码中,因此,给类型构造器和值构造器取相同的名字是被允许的.

类型别名

type ID = Int
type BookName = String
type Author = [String]

data BetterBookInfo = BetterBook ID BookName Author deriving (Show)

可以看到给类型取别名后,新的BetterBookInfo类型具有更好的可读性.但是别名并没有声明了新类型,创建值得过程还是由原来的值构造器进行,因此在可以使用==进行比较(不同类型之间是不可比的).

代数数据类型

代数数据类型借鉴了代数学中的概念,通常代数数据结构类型不包含任何行为,它利用和类型来展示相同抽象概念的不同组合,使用积类型来展示同一概念不同属性的组合.

关于haskell是否具有真实的代数数据类型我们这里暂不考虑(涉及比较高深的理论),这里只是借此梳理一下相关内容,认为haskell有'代数数据类型'.

最简单的代数数据类型是布尔类型

data Bool = False | True

布尔数据类型拥有两个值构造器,一个是True,另一个是False,两者用|符号分割,可以读作'或'.当一个类型拥有一个以上的值构造器时,这些值构造器通常被称为"备选"(alternatives)或"分支"(case).

一个较为复杂的示例为:

type CardHolder = String
type CardNumber = String
type Address = [String]

data BillingInfo = CreditCard CardNumber CardHolder Address
| CashOnDelivery
| Invoice CustomerID
deriving (Show)

该类型定义了三种付款方式,信用卡付款,支付现金,货到付款.

模式匹配

对于某个类型,应该可以做到以下两点:

  • 如果这个类型有一个以上的值构造器,那么应该可以知道,这个值由哪个构造器创建的
  • 如果一个值构造器包含不同的成分,那么应该有办法提取这些成分

Haskell解决这两个问题的方法,那就是模式匹配.

-- 这是一个脚本,myNot.hs
myNot True = False
myNot False = True

在这里,myNot 不是被定义了两次,而是定义了两个模式.在每次调用myNot函数时,首先需要对域进行模式匹配,再返回相应的值.如果匹配到True则返回False,如果匹配到False则返回True,匹配到其他的内容将报错.

构造与解构

当我们构造一个值时,例如表达式Book 123 "haskell" ['abc']将Book构造器应用到123,"haskell",['abc']上面,从而产生了一个新的BookInfo类型的值.

当我们进行模式匹配的时候,相当于构造的逆过程--解构.
对于同样的例子Book 123 "haskell ["abc"]",对模式Book id name authors进行匹配.

-值构造器与模式中的构造器相同,构造器匹配

  • 变量id被绑定为123,成功
  • 变量name被绑定为haskell
  • 变量authors被绑定为["abc"]

模式匹配中的形参命名

对于myNot这种简单的模式匹配,只需要一个值构造器即可,但如果值构造器后面接受参数,那么就需要考虑参数的命名问题.

--sumList.hs
sumList [] = 0
sumList (x:xs) = x + sumList xs

求列表中数字的和,这里x,xs就是形参,x:xs代表一种模式,即[x,...].

通配符的作用

对于上下文中没有用到的参数,我们可以使用通配符进行标识,如下:

getID (Book id _ _) = id
getBookName (Book _ name _) = title
getAuthors (Book _ _ authors)

类似地,addList可以改成:

-- addList.hs
addList1 (x:xs) = x + addList1 xs
addList1 _ = 0

注意到,addList使用通配符在声明函数定义的位置,_可以匹配任何模式,但是实际上由于haskell是按顺序解析的,因此对于(x:xs)模式,不会匹配到第二个式子,而是被第一个式子"拦截".

如果将两个语句调换顺序,那么第二个语句将永远不会被用到

记录语法

haskell提供了访问数据类型成分的方法,而无需另外书写代码.

data BookInfo = Book {
bookId :: Int,
bookName :: String,
author :: [String]
} deriving (Show)

一旦定义好了,就可以使用定义类型时的参数名称,申城相应的访问函数.

ghci>:type bookId
bookId :: BookInfo -> Int

构造值的方法可以沿用前面提到的方法,当然还可以使用更详细的标识进行构造,以提升代码的可读性,同时由于具有形参标识,我们可以对成分调换顺序

book = Book {
    bookId = 123,
    author = ["abc"],
    bookName = "haskell"
}

如果使用记录语法进行定义,那么在交互界面中的输出格式也会更加详细

ghci>book
Book {bookId = 123, bookName = "haskell", author = ["abc"]}

参数化类型

haskell 中的类型可以通过接收参数从而具有多态的性质.前面提到的列表类型就是多态的,因此列表中的元素才可以是任何类型.

Prelude中定义了一种Maybe的类型

ghci>:info Maybe
data Maybe a = Nothing | Just a         -- Defined in ‘GHC.Maybe’

Maybe类型表达一种可能有值但也可能空缺的情况,例如前面提到的sumList中sumList []被强制设定为0,这似乎并不合理.

-- goodsumList.hs
sumList (x:xs) = x + sumList xs
sumList _ = []

goodsumList (x:xs) = Just (sumList (x:xs))
goodsumList _ = Nothing

通过对sumList进行包装,我们得到了一个合理的goodsumList函数.在接受空列表时,会返回Nothing.

递归类型

从最熟悉的数据类型入手——列表.

data List a = 
    Cons a (List a)
    | Nil
    deriving (Show)

List a在等号两侧都出现了,可以说该类型的定义引用了自己.当使用Cons构造器创建一个值的时候,我们必须提供一个a的值作为参数一,以及一个List a类型作为参数二.

我们平时熟悉的写法[1,2,3],从实际构造角度可以写成Cons 1 ( Cons 2 ( Cons 3 Nil )).

二叉树也是一个比较典型的递归类型.

data Tree a = Node a (Tree a) (Tree a)
    |   Empty
        deriving (Show)

引入局部变量

在函数体内部,可以使用let表达式引入新的局部变量.

--file: Lending.hs
lend amount balance = let reserve = 100
                          newBalance = balance - amount
                    in if balance < reserve
                    then Nothing
                    else Just newBalance

let作为变量声明块的开始,in标识区块的结束,每行引入了一个局部变量.

注意:

  1. 在let区块中,变量名被绑定到了表达式上而不是一个值.这是由于haskell本身惰性求值的特性决定的,即变量名所绑定的表达式一直到被用到时才会求值.
  2. 注意上面代码中let区块中每行引入变量的缩进对齐,如果缩进不当会造成报错(该问题将在后面一个小结着重讲解)

一般地,将代码可以使用一个变量名的地方乘坐这个变量名的作用域(scope).在域内,可以使用这个变量名,否则不能.如果一个变量名在整个源代码的任意处都可以使用,则说明它位于最高层的作用域.

where 从句

另外一种方法引入局部变量是where从句,where从句中的定义在其所跟随的主句中有效.

-- file: ch03/Lending.hs
lend2 amount balance = if amount < reserve * 0.5
                       then Just newBalance
                       else Nothing
    where reserve    = 100
          newBalance = balance - amount

与let表达式一样,where从句中的空白和缩进也很重要.

局部函数与全局变量

在Haskell语法中,定义变量和定义函数的方式非常相似.

定义全局变量:

global_string = "haskell"

定义局部函数:

pluralise :: String -> [Int] -> [String]
pluralise word counts = map plural counts
    where plural 0 = "no " ++ word ++ "s"
          plural 1 = "one" ++ word
          plural n = show n ++ " " ++ word ++ "s"

表达式中的越位规则和空白

Haskell使用缩进作为解析代码段的提示。这种使用布局来传达结构有时被称为越位规则。在在源文件开始时，第一个顶层声明或定义可以从任何列开始，并且Haskell编译器或解释器记住这个缩进级别。每个后续的顶级声明必须具有相同的缩进。

let和where表达式是相似的,在关键字后,编译器或解释器会记住下一个标记的缩进,如果下一行是空行,或者缩进更向右,则被认为是上一行的延续;如果缩进与上一项相同,那么这一行将被当成同一个代码块中一个新项的开始.

例如:

foo = let a = 
               "haskell"
          b = "interesting"
      in  a ++ " " ++ b

一点提示: 由于某些系统原因,需要将tab更换为一定数量的space,否则会引起编译问题

然而haskell中越位规则并非强制性的,可以使用花括号和分号来显式表达语句.

foo = let {a = "haskell" ; b = "insteresting"}
    in a ++ " " ++ b

当使用显示结构后,原来的越位和缩进规则将不再生效.

熟悉C语言和python的读者可以发现,越位规则中的缩进类似python语法风格,而显式结构则类似C风格

case 表达式

函数定义并不是我们可以使用模式匹配的唯一地方.case构造允许我们匹配表达式中的模式.

例如前面的myNot定义可以写为:

myNot a = case a of
    True -> False
    False -> True

注意到示例使用了越位规则,对两种模式进行了缩进对齐,这当然可以写成:

myNot a = case as of
    { True -> False ; False -> True}

嵌套组合:

myAnd a b = case a of
                False -> False
                True -> case b of
                        False -> False
                        True -> True

也可以写成元组的模式匹配:

myAnd a b = case (a,b) of 
                (True,True) -> True
                _ -> False
