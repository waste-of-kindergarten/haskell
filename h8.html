正则表达式

正则表达式描述了一种字符串匹配的模式,可以用来检查一个串是否含有某种子串,将匹配的子串替换或从某个串中取出符合某个条件的子串等.

为了使用正则表达式,我们需要regexp库.

ghci>:module +Text.Regex.Posix

在我的haskell解释器中,并没有找到这个模块,于是我选择用cabal进行安装.

cabal install --lib regex-posix

正则表达式匹配函数可能是我们需要用到的唯一的函数,以中缀运算符=~表示.haskell中的正则表达式与其他语言类似,唯一不同的是haskell的regexp库使用了多态,这导致了=~类型签名非常晦涩,因此我们只讨论应用.

普通字符

普通字符包括所有大写和小写字母,数字,标点符号和一些其他符号.

匹配集合

[...]表示匹配方括号中的全部字符,[^...]表示匹配除了方括号中全部字符.

ghci>"waste of kindergarten" =~ "[t]" :: Int
2
ghci>"waste of kindergarten" =~ "[wt]" :: Int
3
ghci>"waste of kindergarten" =~ "[gw]a" :: Int
2
ghci>"waste of kindergarten" =~ "[^g]a" :: Int
1
ghci>"waste of kindergarten" =~ "[^g]a" :: String
"wa"
ghci>"waste of kindergarten" =~ "[^g]a" :: (String,String,String)
("","wa","ste of kindergarten")

可以看到,我们必须强制声明匹配函数返回的类型才能得到想要的结果,通过不同的类型签名,可以得到不同的结果,如Int可以得到匹配的子串数量,String得到第一个匹配的子串,(String,String,String)可以得到匹配子串及其前后的字符串(前面的字符串,匹配子串,后面的字符串).

如果需要匹配的集合元素很多,可以使用推导式,如[A-Z]表示"AB...Z"一共26个大写字母.

ghci>"waste of kindergarten" =~ "[a-z]" :: Int
19
-- 总长为21,匹配过程排除了2个空格

匹配换行符与空白符

\s匹配所有空白符和换行符;\S匹配非空白符,非换行符;.匹配所有非换行符.

ghci>"waste of\nkindergarten" =~ "\\s" :: String
" "
ghci>"waste\rof\nkindergarten" =~ "\\s" :: String
"\r"
ghci>" waste of kindergarten" =~ "\\S" :: String
"w"
ghci>"\nwaste of kindergarten" =~ "\\S" :: String
"w"
ghci>" waste of kindergarten" =~ "." :: String
" "
ghci>"\nwaste of kindergarten" =~ "." :: String
"w"

当然,如果要匹配换行符或空格可以直接用原始字符进行匹配.

ghci>"\nhaskell" =~ "\n" :: Bool
True
ghci>"\rhaskell" =~ "\r" :: Bool
True
ghci>"\thaskell" =~ "\t" :: Bool
True

匹配字母,数字与下划线

\w可以一次性匹配所有字母,数字和下划线,与[A-Za-z0-9_]是等价的.

ghci>"waste of kindergarten" =~ "\\w" :: Int
19

特殊字符

特殊字符指的一些具有特殊含义的字符.如果要匹配特殊字符本身含义,则需要(双)反斜杠来进行转义.

'+'

+代表该符号前面的字符至少出现一次.

ghci>"hello haskell" =~ "hello+" :: String
"hello"
ghci>"hellooo haskell" =~ "hello+" :: String
"hellooo"

'*'

*代表该符号前面的字符可能出现,即不出现 或者 出现一次或多次.

ghci>"hello haskell" =~ "hello*" :: String
"hello"
ghci>"hellooo haskell" =~ "hello*" :: String
"hellooo"
ghci>"hell haskell" =~ "hello*" :: String
"hell"

'?'

?代表至多出现一次,即不出现或者出现一次.

ghci>"hello haskell" =~ "hello?" :: String
"hello"
ghci>"hell haskell" =~ "hello?" :: String
"hell"
ghci>"helloo haskell" =~ "hello?" :: String
"hello"

'$'

$匹配字符串的结尾位置,这个结尾包括'\n'(windows下不匹配'\r').

ghci>"hello haskell" =~ ".*$" :: String
"hello haskell"
ghci>"hello\rhaskell" =~ ".*$" :: String
"hello\rhaskell"
ghci>"hello\nhaskell" =~ ".*$" :: String
"hello"

'^'

^匹配字符串的开始位置(注意使用范围为方括号表达式外,否则代表的含义如上).

ghci>"hello hello" =~ "^hello" :: (String,String,String)
("","hello"," hello")

'()'

()标记一个子表达式开始和结束的位置.

'|'

|指明两项之间选择一个.

ghci>"hello holle" =~ "(e|o)ll(e|o)" :: Int
2

'[]'

[]标记中括号表达式.

'{}'

{}标记限定符表达式.

限定符

限定符用来指定正则表达式的一个给定组件必须要出现多少次才能匹配.一共有* + ? {n} {n,} {n,m}6种限定符,其中前三种已经在前面提到,这里不再赘述.

'{n}'

n表示一个非负整数,匹配确定的n次.

ghci>"hello" =~ "hel{2}o" :: Bool
True
ghci>"hello" =~ "hel{1}o" :: Bool
False

'{n,}'

n表示一个非负整数,至少匹配n次.

ghci>"hello" =~ "hel{1}o" :: Bool
True
ghci>"hello" =~ "hel{2}o" :: Bool
True
ghci>"hello" =~ "hel{3}o" :: Bool
False

'{n,m}'

n<=m均为非负整数,匹配次数在[n,m].

ghci>"hellooooo" =~ "hello{2,4}" :: String
"helloooo"
ghci>"hellooo" =~ "hello{2,4}" :: String
"hellooo"
ghci>"hello" =~ "hello{2,4}" :: String
""

不难看出*等价于{0,},+等价于{1,},?等价于{0,1}.

定位符

定位符使正则表达式能够定位.共有^ $ \b \B4种定位符,其中前两个定位符已经提到.

'\b'

匹配一个单词边界(字与空格).

ghci>"hello world" =~ "\\bworld" :: String
"world"

'\B'

匹配一个非单词边界.

ghci>"hello world" =~ "hell\\B" :: String
"hell"
