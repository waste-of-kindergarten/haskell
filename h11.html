错误处理

由于haskell非常注重纯度,因此错误处理也分为"纯"错误处理和"非纯"错误处理(异常)."纯"的错误处理,指算法发不依赖任何IO Monad,而异常则只会在IO Monad中被捕获.

基于数据类型的错误处理

使用Maybe

如果你熟悉haskell,那么首先想到的应该是Maybe类型,在之前的笔记中,我们已经多次使用过Maybe表示异常的情况,如对于除法分母为0等等.

divBy :: Integral a => a -> [a] -> Maybe [a]
divBy _ [] = Just []
divBy _ (0:_) = Nothing
divBy numerator (denom:xs) =
	case divBy numerator xs of
		Nothing -> Nothing
		Just results -> Just ((numerator `div` denom) : results)

这里使用Nothing来表示处理错误,如果结果正常,则返回Just 结果列表.

ghci>divBy 50 [1,2,3,4]
Just [50,25,16,12]
ghci>divBy 50 [1,2,0,4]
Nothing

惰性丢失与保存

按照以上的使用方法,可以很方便地解决错误的输入,但是却失去了惰性.对于无穷链表,divBy必须遍历完全部元素,才能决定是Just [a]还是Nothing.这将导致惰性丢失,例如,对于普通的列表,我们可以:

ghci>ls = [1..]
ghci>head ls
1

但是考虑divBy 50 [1..],我们甚至都不知道它的模式是Just [a]还是Nothing(从理论上需要无穷次操作),更不能对其进行下一步操作.另外,这种方法对于比较大的有限列表也不是很友好——它的空间效率不高.

注意到上面的divBy的错误只是列表中的某个元素,因此我们可以使用一种取代方法a -> [a] -> [Maybe a].

divBy1 :: Integral a => a -> [a] -> [Maybe a]
divBy1 numerator denominators = 
	map worker denominators
	where worker 0 = Nothing
		  worker x = Just (numerator `div` x)

通过这种改进,我们保留住了惰性,并能够看到哪里出了问题.

ghci>divBy1 50 [1,2,3,4]
[Just 50,Just 25,Just 16,Just 12]
ghci>divBy1 50 [1,2,0,4]
[Just 50,Just 25,Nothing,Just 12]

Maybe Monad 用法

上面提到的divBy函数可以写成monadic风格.

divBy :: Integral a => a -> [a] -> Maybe [a]
divBy _ [] = return []
divBy _ (0:_) = fail "division by zero in divBy"
divBy numerator (denom:xs) = 
	do next <- divBy numerator xs
		return ((numerator `div` denom) : next)

具体有关Monad的用法将在下次笔记具体说明.

使用Either

Either类型与Maybe类型类似,定义如下:

data Either a b = Left a | Right b

Either 于Maybe不同的是，Either无论错误还是正确，都可以携带数据.按照惯例,Left表示错误,Right表示成功.我们使用Either重写一下divBy函数.

divBy :: Integral a => a -> [a] -> Either String [a]
divBy _ [] = Right []
divBy _ (0:_) = Left "divBy: division by 0"
divBy numerator (denom:xs) = 
	case divBy numerator xs of
		Left x -> Left x
		Right results -> Right ((numerator `div` denom) : results)

测试结果如下

ghci> divBy 50 [1,2,3,4]
Right [50,25,16,12]
ghci> divBy 50 [1,2,0,4]
Left "divBy: division by 0"

定义错误类型

除了使用字符串表示错误,我们还可以自定义错误类型,通过自定义错误类型,我们可以很容易地区分多种不同的错误.

为了说明问题,我们沿用前面的divBy,人为添加约束,假设被除数不允许是10的倍数.

data DivByError a = DivBy0
                | ForbiddenDenominator a 
                deriving (Eq,Read,Show)
divBy :: Integral a => a -> [a] -> Either (DivByError a) [a]
divBy _ [] = Right []
divBy _ (0:_) = Left DivBy0
divBy numerator (denom:xs) 
    |  denom `mod` 10 == 0 = Left (ForbiddenDenominator denom)
    | otherwise = case divBy numerator xs of
        Left x -> Left x
        Right results -> Right ((numerator `div` denom) : results)

测试结果如下

ghci> divBy 50 [1,2,3,4]
Right [50,25,16,12]
ghci> divBy 50 [1,2,0,4]
Left DivBy0
ghci> divBy 50 [1,2,30,4]
Left (ForbiddenDenominator 30)

异常

在Haskell中,某些问题尤其涉及到IO调用,需要处理异常,这种异常可能在任何地方跑出,但只可以在IO monad中捕获.

try

在Control.Excetion模块中,定义了各种与异常相关的函数和类型.例如try函数,这个函数在其他语言(如C语言,python等)是常用的,它在Hasell中定义与使用如下:

ghci> :m Control.Exception
ghci Control.Exception> x = 1 `div` 0
ghci Control.Exception> y = 3 `div` 1
ghci Control.Exception>x
*** Exception: divide by zero
ghci Control.Exception> y
3
ghci Control.Exception>:type try
try :: Exception e => IO a -> IO (Either e a)
ghci Control.Exception>try (print x) :: IO (Either ArithException ())
Left divide by zero
-- 也可以使用 SomeException 类型,它为所有异常的继承类型
ghci Control.Exception>try (print y) :: IO (Either ArithException ())
3
Right ()

提示:real world教程在这个地方的代码示例我并没有完全跑通,如果不对try (print x)后面添加类型声明(IO (Either ArithException ())或``),那么它将无法被打印出来,在语句后面添加类型是我找到的修复方案.如果有更好的方案希望您不吝赐教.

由于惰性求值,"1 div 0"并没有直接抛出异常,而是将内容保存在变量x中,只有在后面try (print x)时才需要计算x;另外,可以看到try (print y)输出有两行,第一行是print y的结果,第二行由ghci生成,用于表示print y的返回值为()并且没有抛出异常.

惰性和异常处理

掌握了try的用法后,如果我们需要捕获try的结果用于后续处理,尝试这样做:

ghci Control.Exception> result <- (try (return x) :: IO (Either ArithException Integer))
ghci Control.Exception> result
Right *** Exception: divide by zero

其中有一个奇怪的地方,将结果保存在result中后,打印result却发现它是Right ...,这显然不是我们想要的结果.导致这样的结果的原因,是return本身也具有惰性,即它仅仅对参数进行包装,而不会执行参数.因此,包装过程不会产生异常,从而也就产生了Right ...的结果.想比之下,上面的try (print x)代码则可以正确输出异常,因为打印x要求必须执行x.

为了避免这种情况,可以使用另一个函数evaluate,他的行为与return类似,但是会让参数立即执行.

ghci Control.Exception> result <- (try (evaluate x) :: IO (Either ArithException Integer))
ghci Control.Exception> result
Left divide by zero

对于纯代码中抛出异常的捕获,切记使用evaluate而不是return.

使用handle

以上的异常捕获在异常不发生时,返回一个计算值.如果想在没有异常发生时,执行某个动作,可以使用handle函数.

ghci Control.Exception> :type handle
handle :: Exception e => (e -> IO a) -> IO a -> IO a

handle函数接收两个参数,第一个参数是一个一元函数,第二个参数是动作,当第二个参数动作发生异常时,则调用第二个函数(此时执行另一个动作).

ghci Control.Exception> handle ((\_ -> putStrLn "Error : divide by zero") :: ArithException -> IO ()) (print x)
Error : divide by zero
-- handle ((\_ -> putStrLn "Error : divide by zero") :: SomeException -> IO ()) (print x)
ghci Control.Exception>handle ((\_ -> putStrLn "Error : divide by zero") :: ArithException -> IO ()) (print y)
2

选择性处理异常

上面的异常捕获对于任何异常都是相同的动作处理,我们想要对于不同的异常给出不同的动作,handleJust函数就可以满足这种需要.

ghci Control.Exception>:type handleJust
handleJust :: Exception e => (e -> Maybe b) -> (b -> IO a) -> IO a -> IO a

import Control.Exception
catchIt :: ArithException -> Maybe ()
catchIt DivideByZero = Just ()
catchIt _ = Nothing

handler :: () -> IO ()
handler _ = putStrLn "Caught error : divide by zero"

safePrint :: Integer -> IO ()
safePrint x = handleJust catchIt handler (print x)

catchIt定义了一个函数,用于抓取感兴趣的异常类型,比如上面代码我们关注DivideByZero,因此在函数接受到这个异常时,返回Just (),其他的异常则返回Nothing.handler为关注的异常发生时执行的动作.safePrint x执行打印x的动作,当执行产生异常时,通过catchIt判断是否感兴趣,如果是则执行handler的动作,否则不执行.

实际上,catchIt返回的Just ()中附带的()会被传到handler中,注意到异常信息并没有被传递到handler,此时对于所有关注的异常,执行的动作是相同的,这显然不是我们想要的结果.

import Control.Exception

catchIt :: ArithException -> Maybe ArithException
catchIt e = Just e

handler :: ArithException -> IO ()
handler e = putStrLn $ "Caught arithmetic error: " ++ show e

safePrint :: Integer -> IO ()
safePrint x = handleJust catchIt handler (print x)

在ghci中试验一下:

ghci Control.Exception>safePrint (1 `div` 0)
Caught arithmetic error: divide by zero
ghci Control.Exception>safePrint (5 `div` 2)
2

抛出异常

Control.Exception中最通用的抛出异常的函数是throw.

ghci Control.Exception>:type throw
Exception e => e -> a

这个函数可以抛出任何Exception,并可以用于纯的上下文中,即任何Exception的实例都可以被抛出.下面给出一个示例:

data MyException = MyException1
    | MyException2
    
instance Exception MyException

instance Show MyException where
    showsPrec _ MyException1 = showString "My first Exception"
    showsPrec _ MyException2 = showString "My second Exception"
    
div1 :: Integer -> Integer -> Integer
div1 a b 
    | b == 0 = throw MyException1
    | otherwise = a `div` b

ghci>throw MyException1
*** Exception: My first Exception
ghci>throw MyException2
*** Exception: My second Exception
ghci>div1 1 0
*** Exception :: My first Exception
ghci>:type evaluate (div1 1 0)
evaluate (div1 1 0) :: IO Integer
ghci>try $ evaluate (div1 1 0) :: IO (Either MyException Integer)
Left My first Exception

另外一个用于在IO monad中抛出异常的函数是throwIO.

ghci Control.Exception>:type throwIO
throwIO :: Exception e => e -> IO a

沿用MyException的一个示例:

safePrintdiv :: Integer -> Integer -> IO ()
safePrintdiv x y 
    | y == 0 = throw MyException1
    | otherwise = print (x `div` y)

ghci>safePrintdiv 1 2
0
ghci>safePrintdiv 1 0
*** Exception: My first Exception
ghci>:type (safePrintdiv 1 0)
(safePrintdiv 1 0) :: IO ()
ghci>try (safePrintdiv 1 0) :: IO (Either MyException ())
Left My first Exception

还有一个函数ioError位于System.IO.Error和Control.Exception,其定义是完全相同的,其类型为IOError -> IO a,可以用于生成任意I/O相关的异常.
