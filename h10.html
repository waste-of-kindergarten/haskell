数据类型

字典

Haskell中如何表示字典之类的数据结构呢?最常用的是关联列表和Data.Map模块提供的Map类型.

关联列表

关联列表是一种特殊的列表类型,类型签名为[(a,b)].关联列表内部的元素(a,b)实际上就是字典的键-值对.

ghci>students = [(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]
ghci>:type lookup
lookup :: Eq a => a -> [(a, b)] -> Maybe b
ghci>lookup 12345 students
Just "Bob"
ghci>lookup 0 students
Nothing
-- lookup 函数可以根据键查找关联列表对应的值

Map类型

Data.Map模块提供的Map类型的行为和关联列表类似,但Map的性能更好.

注意:由于Data.Map中有函数与Prelude中的函数重名,可以通过import qualified Data.Map as Map的方式引入模块,当需要使用Data.Map中的函数时,可以通过Data.Map.函数名 来调用.

ghci>import qualified Data.Map as Map
ghci Map>students = [(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]
ghci Map>:type Map.fromList
Map.fromList :: Ord k => [(k, a)] -> Map.Map k a
ghci Map>studentsMap = Map.fromList students
-- fromList [(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]
ghci Map>:type Map.insert 
Map.insert :: Ord k => k -> a -> Map.Map k a -> Map.Map k a
ghci Map>studentsMapFold = foldl (\map (k,v) -> Map.insert k v map) Map.empty students
-- fromList [(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]
ghci Map>studentsMapManual = Map.insert 12345 "Bob" . Map.insert 13579 "Alfred" . Map.insert 24680 "Adam" $ Map.empty 
-- fromList [(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]
ghci Map>Map.toList studentsMap
[(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]

实例中给出了三种方式将关联列表转换为Data.Map.Map数据类型——fromList函数转换数据类型;foldl递归插入键值对;利用复合运算符串联手动插入键值对.

注意:Map不能保证它的输出排列和原本 的输入排列一致.

下面介绍一些常用的操作函数.

  • 运算符

(!)

根据键查找Map中值,如果没有找到,则会调用error.

ghci Map>:info Map.!
(Map.!) :: Ord k => Map.Map k a -> k -> a
        -- Defined in ‘Data.Map.Internal’
ghci Map>studentsMap Map.! 12345
"Bob"
ghci Map>studentsMap Map.! 0
"*** Exception: Map.!: given key is not an element in the map
CallStack (from HasCallStack):
  error, called at libraries\\containers\\Data\\Map\\Internal.hs:610:17 in containers-0.6.0.1:Data.Map.Internal

当然,Map也含有一个lookup函数,返回的是Maybe类型,当没有查到时返回Nothing.

(\\)

差集,与difference相同.

ghci> studentsMap Map.\\ studentsMap
fromList []
ghci>studentsMap Map.\\ (Map.fromList [(0,123)])
fromList [(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]
  • 查询

null

查看Map是否是空.

ghci Map>:info Map.null
Map.null :: Map.Map k a -> Bool         -- Defined in ‘Data.Map.Internal’
ghci Map>Map.null Map.empty
True
ghci Map>Map.null studentsMap
False

size

查看元素个数.

ghci Map>Map.size Map.empty
0
ghci Map>Map.size studentsMap
3

member

查看某个键是否是元素.

ghci Map>Map.member 12345 studentsMap
True
ghci Map>Map.member 0 studentsMap
False

相反的,还有notMember函数.

lookup

根据键查找值，返回Maybe类型.

ghci Map>Map.lookup 12345 studentsMap
Just "Bob"
ghci Map>Map.lookup 0 studentsMap
Nothing

findWithDefault

根据键查找值，如果没有找到返回默认值.

ghci Map>Map.findWithDefault "nobody" 12345 studentsMap 
"Bob"
ghci Map>Map.findWithDefault "nobody" 0 studentsMap 
"nobody"
  • 构造

empty

ghci Map>Map.empty
fromList []

singleton

构造一个只有一个键值对的字典.

ghci Map>Map.singleton 1 "abc"
fromList [(1,"abc")]
  • 插入

insert

插入键值对.

ghci Map>Map.insert 1 "abc" Map.empty
fromList [(1,"abc")]
ghci Map>Map.insert 1 "abd" (Map.fromList [(1,"abc")])
fromList [(1,"abd")]
-- 如果键存在则更新值

insertWith

insertWith 比较复杂，其签名为insertWith :: Ord k => (a -> a -> a) -> k -> a -> Map k a -> Map k a.

该函数接受四个参数，第一个参数是一个函数，第二个是键，第三个是值，最后一个是Map.如果待插入的键在Map中没有出现过，则正常插入；如果待插入的键存在，则对该键的值和Map中的值使用第一个参数（a->a->a），将结果作为新的值。

ghci Map>Map.insertWith (++) 0 "Stu." studentsMap
fromList [(0,"Stu."),(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]
ghci Map>Map.insertWith (++) 12345 "Stu." studentsMap
fromList [(12345,"Stu.Bob"),(13579,"Alfred"),(24680,"Adam")]

insertWith'

效果与insertWith相同，但是是严格求值的.

insertWithKey

顾名思义，效果与insertWith类似，但是更新值得时候多了一个参数是Key.签名insertWithKey :: Ord k -> (k -> a -> a -> a) -> k -> a -> Map k a -> Map k a,其中第一个参数是一个更新函数，这个函数接受Key,newValue,oldValue,并将运算的结果作为Map中新的值.

insertWithKey'

效果与insertWith相同，但是是严格求值.

insertLookupWithKey

包含了insertWithKey的功能，并增加了查找的功能。即当键在Map中时，返回一个Just 旧值和并更新旧值为新值；否则，返回Nothing,并更新Map中的键值对.签名insertLookupWithKey :: Ord k => (k -> a -> a -> a) -> k -> a -> Map k a -> (Maybe a , Map k a).

insertLookupWithKey'

insertLookupWithKey的严格求职版本。

  • 删除和更新

delete

根据键删除一个键值，如果不存在则返回原来的Map.

ghci Map>Map.delete 12345 studentsMap
fromList [(13579,"Alfred"),(24680,"Adam")]

adjust

更新键值对，当键不存在的时候，返回原Map.

ghci Map>Map.adjust ("new" ++) 12345 studentsMap
fromList [(12345,"newBob"),(13579,"Alfred"),(24680,"Adam")]
ghci Map>Map.adjust ("new" ++) 0 studentsMap
fromList [(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]

adjustWithKey

更新函数接受键作为其参数.签名为adjustWithKey :: Ord k => (k -> a -> a) -> k -> Map k a -> Map k a.

update

签名如下：

update :: Ord k => (a - > Maybe a) -> k -> Map k a -> Map k a.函数在Map中查找k，如果有，那么将其值进行更新(a -> Maybe a),如果更新函数结果是Nothing则直接删除该键值;如果k没在Map中，则不进行更新.

ghci Map>Map.update (\ x-> if x== "Bob" then Just "new Bob" else Nothing) 12345 studentsMap
fromList [(12345,"new Bob"),(13579,"Alfred"),(24680,"Adam")]
ghci Map>Map.update (\ x-> if x== "Bob" then Just "new Bob" else Nothing) 0 studentsMap
fromList [(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]
ghci Map>Map.update (\ x-> if x /= "Bob" then Just "new Bob" else Nothing) 12345 studentsMap
fromList [(13579,"Alfred"),(24680,"Adam")]

updateWithKey

签名如下：

updateWithKey :: Ord k => (k -> a -> Maybe a) -> k -> Map k a -> Map k a.

更新函数接受键作为参数.

updateLookupWithKey

签名如下：

updateLookupWithKey :: Ord k => (k -> a -> Maybe a) -> k -> Map k a -> (Maybe a, Map k a).

alter

签名如下:

alter :: Ord k => (Maybe a -> Maybe a) -> k -> Map k a -> Map k a.

用来对键所对应的值进行替换，如果键不存在，则向Map中添加新的键值；如果更新函数更新的结果为Nothing,则删除这个键值对（对新增键和旧键同样适用）.

ghci Map> Map.alter (\ x -> Nothing) 0 studentsMap
fromList [(12345,"Bob"),(13579,"Alfred"),(24680,"Adam")]
ghci Map> Map.alter (\ x -> Nothing) 12345 studentsMap
fromList [(13579,"Alfred"),(24680,"Adam")]
ghci Map> Map.alter (\ x -> Just "new") 12345 studentsMap
fromList [(12345,"new"),(13579,"Alfred"),(24680,"Adam")]

更多相关操作函数可以访问官网查询。

函数亦数据

haskell中函数也是一种数据类型，我们可以在定义带有函数的数据结构。

例如我们可以定义一个带有名字和描述的二元运算符。

data Binaryfunc a = Binaryfunc {
		name :: [Char],
		description :: [Char],
		operator :: a -> a -> a
} 

func :: Num a => Binaryfunc a
func = Binaryfunc "plus" "add two Num" plus

func定义了一个加法运算符.

注意func应该显示声明签名，否则系统（ghci）将会自动推导为func :: Binaryfunc Integer,此时func将不能用于如浮点数的运算.

将函数作为数据使用的一个应用就是差异列表.

haskell原生的列表性能相比命令式语言是较差的，为了保证数据的纯度，每次进行列表操作都是生成一个新列表。如拼接：

(++) :: [a] -> [a] -> [a]
(x:xs)  ++ ys = x : xs ++ ys
_ ++ ys = ys

从定义中可以看到创建新列表的开销取决于第一个列表，对于多次拼接来说，总开销将正比于最终列表长度的平方。

ghci> :info (++)
(++) :: [a] -> [a] -> [a]
infixr 5 ++

haskell中拼接的定义是右结合的，它希望左侧的操作数始终保持最短，但是当重复向列表末尾拼接破坏了这种结合性。相比之下，命令式语言则重复拼接的开销只取决于操作的次数。

这是一个严重的性能问题，但是通过将列表写成差异列表的方式，我们可以有效缓解这一问题。

对于一个字符串“abcd”,我们可以使用部分函数来存储这些字符串，即：

ghci>str = ("a" ++) . ("b" ++) . ("c" ++) . ("d" ++)
ghci>:type str
str :: [Char] -> [Char]
ghci>str []
"abcd"

利用部分函数存储字符串，只需一个空列表即可得到字符串，但在未提供空列表之前，他们没有进行真正意义上的拼接，只有提供空列表之后，整个表达式才会不断地使用右结合进行拼接运算。

因此，从惰性的角度来看，这一应用使得开销关于部分应用是线性的，而不是平方。

为了更好地应用差异列表，需要将列表的细节隐藏起来（如++和.）.

首先创建一个DList类型，由于是对现有类型定义一个不同的身份，所以采用newtype.

newtype DList a = DL {
	unDL :: [a] -> [a]
}

unDL是析构函数，它把DL构造器删除掉。

下面再定义append函数：

append :: DList a -> DList a -> DList a
append xs ys = DL (unDL xs . unDL ys)

另外一种等价的写法是：

append' :: DList a -> DList a -> DList a
append' (DL xs) (DL ys) = DL (xs . ys)

为了方便，我们需要在DList和普通列表之间转换.

fromList :: [a] -> DList a
fromList xs = DL (xs ++)

toList :: DList a -> [a]
toList (DL xs) = xs []

另外我们可以根据普通类表,定义一些常用的列表操作:

empty :: DList a
empty = DL id

-- 类似列表中的(:)
cons :: a -> DList a -> DList a
cons x (DL xs) = DL ((x:) . xs)
infixr `cons`

dfoldr :: (a -> b -> b) -> b -> DList a -> b
dfoldr f x xs = foldr f z (toList xs)

需要注意的是,虽然然DList使得拼接很廉价,但某些列表操作是不容易实现的.例如,列表的head函数具有常数开销,二对应的DList则需要先将整个DList转化为普通列表,因此它比普通列表更低效,其开销正比于DList所需的拼接次数.

safeHead :: DList a -> Maybe a
safeHead xs = case toList xs of
	(y:_) -> Just y
	_ -> Nothing

对应普通列表中的map函数,我们可以将差异列表声明为Functor(函子)的实例.

dmap :: (a -> b) -> DList a -> DList k
dmap f = dfoldr go empty
	where go x xs = cons (f x) xs
	
instance Functor DList where
	fmap = dmap

当然我们还可以实现更多的列表操作,当我们实现了足够多的列表操作时,可以通过回到源文件顶部,增加一个模块头.

module DList
(
    DList
    , fromList
    , toList
    , empty
    , append
    , cons
    , dfoldr
 ) where

列表,差异列表,幺半群

抽象代数中,有一类简单的抽象结构——幺半群,由于幺半群满足条件非常低,所以很多数学结构都是幺半群.一个幺半群只需满足以下两个条件:

  • 一个满足结合律的二元运算符.记为(*),即 (a * b) * c = a * (b * c)
  • 一个单位元素.记为e,即a * e = e * a = a

幺半群不要求这个二元运算符做什么,只要求存在即可.例如将整数的加法作为操作符,0作为单位元素,则整数是一个幺半群;把乘号作为操作符,1作为单位元素,则整数又成了另一个幺半群.

Haskell中的幺半群类定义在Data.Monoid模块中,简单的定义如下:

class Monoid1 a where
    mempty1 :: a  -- 单位元
    mappend1 :: a -> a -> a -- 可结合的二元操作符

实际上Data.Monoid(或者GHC.Base)中的Monoid是继承Semigroup(半群)的,所以如果要定义幺半群的实例,必须先定义半群的实例,这里为了简便直接定义的自由幺半群.

如果把列表的拼接作为二元运算符,空列表作为单位元,则列表就是一个幺半群.

instance Monoid1 [a] where
     mempty1 = []
     mappend1 = (++)

DList其实也是一个幺半群.

instance Monoid1 (DList a) where
    mempty1 = empty
    mappend1 = append

注意: 虽然从数学的角度上,整数可以以两种不同的方式作为幺半群,但haskell不能为整数写两个Monoid实例,否则编译器会报错.

一种妥协办法是,使用newtype 创建不同的类型来达到目的,即通过newtype来区分两种不同的幺半群.

{-# LANGUAGE GeneralizedNewtypeDeriving #-}
-- 广义新类型派生,理论上newtype定义的类型和旧类型是等价的,只是它们名义上的类型不同,但是您通常希望新类型的类型类实例与基础类型的实例完全相同.
newtype AInt = A {unA :: Int} deriving (Show,Eq,Num)

instance Monoid1 AInt where
 mempty1 = 0
 mappend1 = (+)

newtype MInt = M {unM :: Int} deriving (Show,Eq,Num)

instance Monoid1 MInt where
 mempty1 = 1
 mappend1 = (*)

另外,如果您想使用Data.Monoid的类型类声明实例的话,可以按照如下代码:

{-# LANGUAGE GeneralizedNewtypeDeriving #-}
import qualified Data.Monoid as M
import qualified Data.Semigroup as S
newtype Int1 = A {unA :: Int} deriving (Show,Eq,Num)~~~~
instance S.Semigroup Int1 where
    (<>) = (+)

instance  M.Monoid Int1 where
    mempty = 0
    --mappend = (+)
    
newtype Int2 = M {unM :: Int} deriving (Show,Eq,Num)~~~~

instance S.Semigroup Int2 where
    (<>) = (*)

instance  M.Monoid Int2 where
    mempty = 1
    --mappend = (*)

通用序列

无论是Haskell内置列表,还是之前定义的差异列表,这些数据结构在不同的地方都又自己的性能短板,为此Data.Sequence模块定义了Seq容器类型,对于大多数操作,这种类型都能提供良好的效率保证.

ghci>import qualifed Data.Sequence as Seq
ghci Seq> Seq.empty
fromList []
ghci Seq>Seq singleton 1
fromList [1]
ghci Seq>Seq.fromList [1,2,3]
fromList [1,2,3]

Data.Sequence模块还提供了几种操作符形式的构造函数.

ghci Seq>import Data.Sequence((><),(<|),(|>))
-- 直接导入,避免写前缀
ghci Seq Data.Sequence>1 <| Seq.singleton 2
fromList [1,2]
ghci Seq Data.Sequence>Seq.singleton 1 |> 2
fromList [1,2]
ghci Seq Data.Sequence>Seq.fromList [1,2] >< Seq.fromList [3,4]
fromList [1,2,3,4]

可以看到,无论从左边添加元素,还是从右边添加元素,添加操作都可以在常数时间内完成.对两个Seq进行append操作也比较廉价,复杂度等同于两个Seq中较短的那个Seq的长度的对数.

当我们需要将Seq转换为列表时,那么就需要Data.Foldable模块的帮助.

ghci Seq Data.Sequence> import qualifed Data.Foldable as Foldable
ghci Seq Data.Sequence Foldable>Foldable.toList (Seq.fromList [1,2,3])
[1,2,3]

Data.Foldable中的fold函数可以对Seq进行fold操作.除外,Data.Sequence提供了大量有用的函数,这些函数均类似列表操作函数.详细内容可参照文档.
