类型和函数

Haskell 类型系统

强类型

当我们说Haskell有一个强类型系统时，我们的意思是该类型系统保证程序不能包含某些类型的错误。这些错误来自于试图编写没有意义的表达式，例如使用整数作为函数。例如，如果一个函数希望处理整数，而我们传递给它一个字符串，Haskell编译器会拒绝它。

我们称遵守语言类型规则的表达式为良好类型。不遵守类型规则的表达式是类型不良的，它会导致类型错误。

Haskell关于强类型的另一个观点是，它不会自动将值从一种类型强制转换为另一种类型。(强制转换也称为铸造或转换。)例如，如果函数需要float类型的形参，C编译器会自动地、静默地将int类型的值强制转换为float类型的值，但在类似的情况下，Haskell编译器会引发编译错误。我们必须通过应用强制转换函数显式地强制类型。

强类型有时确实会使编写某些类型的代码更加困难。例如，用C语言编写低级代码的一种经典方法是给定一个字节数组并对其进行强制转换，以便将字节视为真正复杂的数据结构。这是非常有效的，因为它不需要我们到处复制字节。Haskell的类型系统不允许这种强制。为了获得相同的结构化数据视图，我们需要进行一些复制，这将在性能上付出一些代价。

强类型的巨大好处是，它能在代码中的真正错误引发问题之前捕获它们。例如，在强类型语言中，我们不能在需要整数的地方意外地使用字符串。
——《real world haskell》

静态类型

静态类型系统可以在编译的时候知道每个值和表达式的类型。

    ghci> True && "false"
    <interactive>:1:9: error:
        Couldn't match expected type ‘Bool’ with actual type ‘[Char]’
        In the second argument of ‘(&&)’, namely ‘"false"’
        In the expression: True && "false"
        In an equation for ‘it’: it = True && "false"

用动态类型语言编写的程序需要大量的测试套件，以确保不会发生简单类型错误。测试套件不能提供完整的覆盖:一些常见的任务，比如重构程序使其更模块化，可能会引入新的类型错误，而测试套件可能不会暴露这些错误。

最后，Haskell编译器可以自动推断程序中几乎所有表达式的类型。这个过程称为类型推断。Haskell允许我们显式地声明任何值的类型，但是类型推断的存在意味着这几乎总是可选的，而不是我们必须做的事情。

常见的基本类型

  • Char
  • Bool
  • Int
  • Integer (理论上可以无限大)
  • Double

类型签名

    ghci>:type 'a'
    'a' :: Char
    ghci>'a' :: Char
    'a'
    ghci> [1,2,3] :: [Int]
    [1,2,3]

在“::”后加上值的类型称为类型签名。

函数的应用

在haskell中，内部函数调用只需在函数后面添加参数即可,参数之间用空格分开。

    ghci>odd 3
    True
    ghci>even 3
    False

函数在所有操作符中拥有最高的优先级，因而

    ghci>odd 3 = True
    True

对于复杂的嵌套函数使用，有必要使用括号使其被解析，例如

    ghci> compare (sqrt 3) (sqrt 6)
    LT

列表和元组

列表和元组类型非常常见且有用，这里将继续深入讨论有关列表和元组的使用方法。

  • head : 返回列表中的第一个元素
    ghci>head [1,2,3,4]
    1
    ghci>head "abcd"
    'a'
  • tail : 返回列表的尾部元素
    ghci> tail [1,2,3,4]
    [2,3,4]
    ghci> tail "abcd"
    "bcd"
  • take 和 drop : 截取部分列表
    ghci> take 2 [1,2,3,4]
    [1,2]
    ghci> drop 3 [1,2,3,4]
    [4]

take 表示取列表的前几个元素，所有只有当取的元素个数为非负数才可以，drop 表示丢弃列表的前几个元素（取后面若干元素），同样也需要时非负数作为元素个数的参数。特别的，haskell允许提取或丢弃元素的个数超过列表中元素的数目。

    ghci>take -1 [1,2,3,4]
    <interactive>:31:1: error:
    ? Non type-variable argument
        in the constraint: Num ([a1] -> Int -> [a2] -> [a2])
      (Use FlexibleContexts to permit this)
    ? When checking the inferred type
        it :: forall a1 a2.
              (Num a1, Num ([a1] -> Int -> [a2] -> [a2]),
               Num (Int -> [a2] -> [a2])) =>
              Int -> [a2] -> [a2]
    ghci>take 10 [1,2,3,4]
    [1,2,3,4]

注意：对于列表的实例来说，它的类型为[元素类型]，例如[1,2,3,4]类型为[Int],而"asdf"类型为[Char].

元组是具有固定大小的一组值的集合，每个值允许有不同的类型。

    ghci> a = ("Alfred","male","software engineering",123456)
    ghci>:type a
    a :: Num d => ([Char], [Char], [Char], d)

haskell 没有单元素元组类型，()类型充当由一个零元素组成的元组。

  • fst 和 snd
    ghci> fst (1,'a')
    1
    ghci> snd (1,'a')
    'a'

元组的类型同列表的类型，不同元组中元素的数量，元素的类型及顺序的不同都应当被视为不同类型的元组。

    ghci>: type (False,'a')
    (False,'a') :: (Bool,Char)
    ghci>: type ('a',False)
    (a',False) :: (Char,Bool)

函数类型和纯度

上一节最后一部分写了一个简单的基本，其中使用了lines函数，以之为例，查看该函数的类型。

    ghci>:type lines
    lines :: String -> [String]

回显应该解读为，lines函数的类型签名为 String 到 [String]的类型。

关于纯度

副作用会在系统的全局状态和函数的行为之间引入依赖性。例如，让我们暂时离开Haskell，来考虑一种命令式编程语言。考虑一个读取并返回全局变量值的函数。如果其他一些代码可以修改这个全局变量，那么我们的函数的特定应用程序的结果取决于这个全局变量的当前值。这个函数有一个副作用，即使它从不修改变量本身。
——《real world haskell》

如果一个函数输出只取决于显式的提供的内容，则称之为纯函数，否则将拥有副作用，非纯函数。

real world 中给出的示例是

    ghci>:type readFile
    readFile :: FilePath -> IO String

Haskell的类型系统可以防止意外的将非纯代码和纯代码混合在一起。

函数编写与加载

我们可以在ghci中编写函数并使用，但是ghci只是Haskell的一个子集。

haskell源码以hs文件保存，作为一个简单的例子：

    --file: add.hs
    add a b = a + b

保存为add.hs文件后，用ghci打开即可使用add函数(或者使用:load add.hs)

值得注意的是有时ghci无法找到文件所在位置，此时可以通过:cd /path改变当前路径。

    ghci> :load add.hs
    [1 of 1] Compiling Main             ( add.hs, interpreted )
    Ok, modules loaded: Main.
    ghci> add 1 2
    3

变量？

在haskell中，可以为一个变量赋予一个表达式，并用这个变量代替表达式，不同的是，这个变量一旦绑定就无法再次赋值（我认为倒不如称之为常量）。

    --file: variable.hs
    x = 10
    x = 11

条件判断

使用列表中的两个操作take和drop作为例子，我们可以利用条件判断编写一个自己的drop和take函数

    --file : mytake.hs
    mytake n xs = if n <= 0 || null xs
                then []
                else (head xs): mytake (n-1) (tail xs)

    --file: mydrop.hs
    mydrop n xs = if n <= 0 || null xs
    then xs
    else mydrop (n-1) (tail xs)

以 mytake 函数为例，mytake 需要两个参数，一个是提取元素的数目，另一个是列表，即n和xs,表达式右边是一个条件判断语句，如果n为非正数或列表为空，那么返回一个空列表，否则构建一个列表，这个列表的第一个元素是xs的第一个元素，而后面的元素由对列表尾部提取n-1个元素的表达式决定。显然这是一个递归的表达式，这在haskell中既常见又容易。

注意：定义函数时要保证函数在任何情况下返回值的类型是相同的

惰性求值

haskell的求值具有惰性的特点，在严格求值的语言中，函数的参数在函数被调用前被求值；但haskell采用了非严格求值，即非必要不求值。

我们取一个非常典型的例子：

    ghci> True || length [1..] > 1
    True
    ghci> False && length [1..] > 1
    False

注意到length [1..] > 1 根本无法求值（因为是无穷的），所以可以断定上面的式子中，haskell并没有求这个表达式的值，而是通过第一个参数直接得出值。这是因为任何一个bool类型与True进行或运算都是True,而与False进行与运算都是False,所以没有必要对第二个子式进行求值。这就是惰性求值。

多态

以 head 函数为例，head函数处理的是列表，但正如前文提到的，列表的类型由列表中元素的数量，类型和排列决定，但无论列表式什么类型，head总能取出列表中的第一个元素。

    ghci>:type head
    head :: [a] -> a

head的类型签名中出现了类型变量a，表示参数传入一个列表，列表中的元素类型是a，函数返回一个元素，类型是a.其中a可以是任意类型。

如果一个类型包含类型参数，我们就说它是参数化类型或多态类型。如果一个函数或值的类型包含类型参数，我们称之为多态的。
——《real world haskell》

函数类型的具体含义

前面简单提到过函数类型，但是仅仅从字面上进行了解释，下面从更细致的原理解释。

    ghci>:type take
    take :: Int -> [a] -> [a]

我们知道,take的类型签名可以解读为，take需要两个参数，第一个参数是正数，第二个参数是列表，返回一个列表。但实际上，haskell并不存在接收两个参数的函数，每个函数都只能接受一个参数。

haskell类型签名中,->是右结合的，即从右向左。上面的类型签名也可以写成take :: Int -> ([a] -> [a])——take 接收一个整数，并返回一个新函数，这个新函数接收一个列表并返回一个新列表。

关于纯度的后记

很少有编程语言能像Haskell那样坚持“纯”应该是默认值。这一选择具有深远和宝贵的影响。

因为应用纯函数的结果只能依赖于它的参数，所以我们通常可以通过简单地阅读它的名称并理解它的类型签名来获得纯函数的功能的强烈提示。

纯度使得理解代码的工作更容易。纯函数的行为不依赖于全局变量的值、数据库的内容或网络连接的状态。纯代码本质上是模块化的:每个函数都是自包含的，并具有定义良好的接口。

使用纯性作为默认值的一个不明显的结果是，使用非纯代码变得更容易。Haskell鼓励一种编程风格，将必须有副作用的代码与不需要副作用的代码分开。在这种风格中，不纯代码趋于简单，在纯代码中执行“繁重的工作”。

软件的大部分风险在于与外界沟通，无论是处理不良数据或丢失数据，还是处理恶意攻击。因为Haskell的类型系统准确地告诉我们代码的哪些部分有副作用，所以我们可以适当地保持警惕。因为我们偏爱的编码风格使不纯代码保持隔离和简单，所以我们的“攻击面”很小。
——《real world haskell》
