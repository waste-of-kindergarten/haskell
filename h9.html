Functor 简介

考虑之前学习过的map函数.

ghci>:type map
map :: (a -> b) -> [a] -> [b]

map函数接收一个函数以及一个列表,并将函数作用于列表中的每一个元素并生成新的列表.例如:

ghci>map (2+) [1,2,3,4]
[3,4,5,6]

map的这种行为非常有用,但是它仅仅能处理列表.我们希望得到一个一般化函数,从而对于任意的容器f af \ af a(fff时容器类型,aaa表示多态),函数都能够对容器中的所有的aaa都进行映射a→ba \rightarrow ba→b,最终得到f bf \ bf b.

Haskell提供了一个类型类Functor来一般化map,这个类定义了一个函数fmap.

class Functor (f :: * -> *) where
  fmap :: (a -> b) -> f a -> f b
  (<$) :: a -> f b -> f a

<$ : 等价于 fmap . const ,将容器f b中的所有的b替换为a.但这不是我们讨论的重点.

有了Functor类型类,我们就可以为容器添加实例,例如,一个二叉树:

data Tree a = Empty 
        | Node a (Tree a) (Tree a)
        deriving (Eq,Show)

添加实例:

instance Functor Tree where
    fmap f tree = case tree of
        Empty -> Empty
        Node a lchild rchild -> Node (f a) (fmap f lchild) (fmap f rchild)

任意构建一个二叉树用以验证:

exampletree = Node 1 (Node 2 Empty Empty) (Node 3 Empty Empty)
resulttree = fmap even exampletree
--Node False (Node True Empty Empty) (Node False Empty Empty)

当然对于内置的列表类型,fmap的效果就是map.

如果读者仔细关注定义,则会注意到Functor 后接收的类型(类型构造子),必须满足形如∗→∗* \rightarrow *∗→∗,上面的列表类型以及二叉树类型,均满足这一条件.对于不满足的构造子,则不能声明实例.例如Either a b,(a,b)为∗−>∗−>∗* -> * -> *∗−>∗−>∗,而Int,Bool不能接收类型参数.

除此之外,被声明实例的成员类型的类型参数不能收到约束.例如

data Num a => Cash a = Cash a 
instance Functor Cash where
	fmap f (Cash a) = Cash (f a)

Cash类型接收的类型参数必须是Num的成员.但是这个约束使得我们无法为其声明实例.

考虑fmap接收的第一个参数类型签名为a→ba \rightarrow ba→b,这里的a并没有约束,当声明Cash的实例时,它要求fmap的类型签名加上Num约束.

一种解决方法是在f上加类型约束,而不是在类型上添加约束.

Funtor具有如下两个性质:

  • preserve identity

    fmap id返回完全相同的值,效果与id相同

  • composable

    fmap f1 . fmap f2 与 fmap (f1 . f2)效果相同
